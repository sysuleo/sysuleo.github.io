    [problems solution] windows下 FATAL Port *** has been used排查

    Posted on 2020-12-17 Edited on 2021-04-22 In problems solution Views:

    分析报错

    最近写博客时，遇到用hexo s指令失败报错的问题

    分析该报错，发现端口4000已经被占用

    查找进程

    查找端口号 4000的网络占用

    • -a 显示所有连接和侦听端口。
    • -n 以数字形式显示地址和端口号。
    • -o 显示拥有的与每个连接关联的进程 ID 可以看到端口号 4000被 PID 21144占用

    查找应用

    查找进程号包含 21144的所有进程

    好家伙，就是这个Code.exe占用了

    kill掉应用

    通过查看占用端口号的进程，可以直接杀掉进程（这一步谨慎操作，别杀了不能杀的进程，那就Good Game了）

    • /pid processid 指定要终止的进程的 PID。
    • /f 指定强制终止进程。
    • /t 终止指定的进程和由它启用的子进程
